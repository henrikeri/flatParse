FlatMaster Logic Specification

Normative behavior specification for scanning, matching, materialization, and processing.
Version context: FlatMaster 1.2.x. Updated: 2026-02-27.
1. Scan Logic 2. Exposure Grouping Logic 3. Matching Logic (Priorities and Gates) 4. Manual Override Logic 5. Master Dark Materialization Logic 6. Native Engine Logic 7. PixInsight Engine Logic 8. Output Naming and Format Logic 9. Diagnostics and Confidence Logic

1. Scan Logic

1.1 Combined scan trigger

  • If dark-only mode is disabled, ScanAll requires at least one flat root and one dark root.
  • If roots are on different drives, flat and dark scans run in parallel; otherwise sequentially.
  • After both complete, matching auto-runs once per revision pair.

1.2 Directory traversal rules

  • Traversal is breadth-first with skip directory set: _darkmasters, _calibratedflats, masters, _processed, and dot-prefixed directories.
  • Supported image extensions: .fit, .fits, .xisf.

1.3 Dark library special handling

  • Pre-scan includes potential master dark files and manifest files even under skipped subtrees.
  • Any file under a nearest DARKS### or DARK### ancestor receives DarkGroupFolder reference.
  • Missing master temperatures can be backfilled from sibling darks with same binning using median donor temperature.

2. Exposure Grouping Logic

  1. Flat files are grouped by normalized exposure key.
  2. Each group carries representative metadata, average flat temperature (AverageTemperatureC), and matching criteria snapshot.
  3. Groups with fewer than 3 frames are marked invalid (IsValid = false).

Important: Group temperature in matching UI is the flat-group average temperature, not a single-file sample.

3. Matching Logic (Priorities and Gates)

3.1 Exact/near definitions

  • Exact exposure tolerance: absolute delta < 0.001 s.
  • Near-2s band: (0.001, 2.0] s.
  • Near-10s band: (2.0, 10.0] s.

3.2 Priority waterfall

P1MasterDarkFlat or MasterDarkExact
P2DarkFlat or DarkExact

3.3 Temperature and bias guard semantics

  1. Strict pass filters dark candidates by MaxTempDeltaC if both temperatures exist.
  2. If strict pass fails, relaxed pass can still choose a dark before bias if thresholds allow.
    • Dark-over-bias temperature threshold: DarkOverBiasTempDeltaC
    • Dark-over-bias exposure threshold: DarkOverBiasExposureDeltaSeconds
  3. Selection condition is delta <= threshold for both dimensions when threshold is enabled and data exists.

3.4 Candidate scoring and tie-breaking

Per-candidate match score is based on binning, gain, offset, and temperature proximity contributions. Final deterministic order:

  1. Higher score first
  2. Type priority (master before loose, bias later)
  3. File path lexical order

4. Manual Override Logic

  • Each group row offers override options: Auto, explicit master candidates, and loose-dark group representatives.
  • Override stores ManualDarkPath in group matching criteria when processing starts.
  • If override path is present in catalog, matching returns it immediately with reason prefix Override.

5. Master Dark Materialization Logic

5.1 Normal processing pre-stage

  1. Detect flat groups that selected loose darks (P2/P4/P6 classes).
  2. Map each candidate to top-level DARKS### folder.
  3. Treat all files under that top-level folder as one source pool, ignoring deeper taxonomy names.
  4. Build or reuse a single master dark per required source folder.

5.2 Darks-only mode

  1. Select raw darks only (Dark/DarkFlat).
  2. Group by rounded exposure (0.001s), binning, and rounded gain (0.001).
  3. Within each group, temperature-aware clustering with configurable tolerance (default 1.0 degC).

5.3 Reuse logic

  • Master can be reused if file + manifest exist and key hash/pipeline/source timestamps are consistent.
  • Pipeline version is enforced to invalidate stale products after algorithmic changes.

6. Native Engine Logic

6.1 Group processing pipeline

  1. Resolve best dark for group.
  2. Load selected dark or bias frame once.
  3. Load flat frames in parallel and subtract dark/bias per pixel.
  4. Per-frame multiplicative normalization by exact median.
  5. Compute EqualizeFluxes factors for rejection stage.
  6. Integrate with stack-size-specific rejection strategy.
  7. Rescale by reference median and write output file.

6.2 Rejection strategy by frame count

Frame countAlgorithm
< 3Average
3-5Percentile clip (low 0.20, high 0.10)
6-15Winsorized sigma clip
> 15Linear-fit clipping

7. PixInsight Engine Logic

7.1 Template split

  • PixInsightTemplate.pjsr: flat calibration + integration workflow.
  • PixInsightTemplate.DARKS.pjsr: dark integration-only workflow.

7.2 Invocation model

  • One folder per invocation is enforced for stability and progress fidelity.
  • Generated scripts are written to temp and executed with automation-mode launch flags.
  • Sentinel files are used as completion contract in detached execution scenarios.

7.3 Calibration rules

  • Flat template may run ImageCalibration for flats.
  • Dark template does not run ImageCalibration; it only integrates dark frames.

8. Output Naming and Format Logic

8.1 Format selector

  • Output format can be XISF or FITS.
  • Effective extension in runtime config is normalized to xisf or fits.

8.2 Path conventions

  • Master darks: OutputRoot/Master/Darks/<seconds>s/<temp>degC/MasterDark_...<ext>
  • Native master flats: MasterFlat_<date>_<filter>_Bin<bin>_<exp>s.<ext>
  • PixInsight master flats: MasterFlat_<date>_<filter>_<exp>s.<ext>

Engine-specific naming differences are intentional in current implementation.

9. Diagnostics and Confidence Logic

  • Matching diagnostics are generated per flat file, then grouped for UI display.
  • Group-level temperature delta is computed against group average flat temperature.
  • For raw-dark priorities (P2/P4/P6), dark temperature display can use average of matched raw-dark set in same top-level dark folder and exposure.
  • Confidence display is percentage based on average candidate score for the group.

If diagnostics look inconsistent, inspect reason strings, selected dark path, and group average temperature source first.
