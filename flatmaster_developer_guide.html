1. Environment Setup

Required tools

  • Windows 10 or 11
  • .NET SDK 8.x
  • Visual Studio 2022 (or VS Code + C# tooling)
  • PixInsight installed for PI engine test runs

Recommended local setup

  • Keep source and dark libraries on SSD or separate physical volumes for scan throughput.
  • Use a dedicated output root for repeatable test runs (for example D:\fmOutput).
  • Pin PixInsight path in appsettings or via UI settings persistence.

2. Build, Test, Run

Restore/build/test

dotnet restore FlatMaster.sln

dotnet build FlatMaster.sln -c Debug

dotnet test FlatMaster.sln -c Debug

Run WPF app

dotnet run --project src/FlatMaster.WPF

Alternative helper script

.\build.ps1 -Run
.\build.ps1 -Test -Run

4. Core Development Workflows

4.1 Add or change UI setting

  1. Add observable property in MainViewModel.cs.
  2. Add binding in MainWindow.xaml.
  3. Persist in MainViewModel.Settings.cs load/save paths.
  4. Flow setting into ProcessingConfiguration if runtime behavior depends on it.
  5. Add or update unit tests for downstream service behavior.

4.2 Change matching behavior

  1. Modify priority or gating in DarkMatchingService.
  2. Update diagnostics text if reason strings changed.
  3. Adjust corresponding tests in DarkMatchingServiceTests.
  4. Re-run full tests and verify UI matching table values remain coherent.

4.3 Change materialization behavior

  1. Modify MasterDarkMaterializer and MasterDarkPathing.
  2. Preserve deterministic naming and output-root-only writes.
  3. Validate both native and PixInsight branches.

4.4 Change processing algorithms

  1. Native path: NativeProcessingService + ImageStackingAlgorithms.
  2. PixInsight path: PixInsightTemplate.pjsr and/or PixInsightTemplate.DARKS.pjsr.
  3. Keep native and PI branches behaviorally aligned where intended.

5. Logic-sensitive Areas

Matching risk areas

  • Strict temperature gate and bias-guard thresholds interact; changing one can alter fallback behavior materially.
  • Reason strings are displayed in UI and logs, and are used in some display heuristics.
  • Manual override path must survive plan rebuild and processing call path.

Materialization risk areas

  • Folder scope: all content under top-level DARKS### is treated as one candidate pool.
  • Manifest reuse key and pipeline version must stay consistent.
  • Output naming/pathing changes can break matching reuse and report accounting.

6. Debugging and Observability

Where to inspect

  • UI log panel for coarse and stage logs.
  • Session log path printed at startup (temp location).
  • Detailed file logger configured in App.xaml.cs.

PixInsight execution details

  • Script files are generated under temp (flatmaster_script_*.js).
  • Engine launches PixInsight with retry argument variants and sentinel-file completion checks.
  • Template choice is driven by job relative directory (__DARKMATERIALIZE__ selects dark template).

Native engine progress

Native logs include load counts, calibration counts, normalization stats, selected rejection algorithm, and write completion lines.

7. Release and Publishing

Self-contained publish (example)

dotnet publish src/FlatMaster.WPF/FlatMaster.WPF.csproj \
  -c Release \
  -r win-x64 \
  --self-contained true \
  /p:PublishSingleFile=true

License files are embedded resources and available at runtime even in self-contained deployments.

Release validation checklist

  1. Build and tests pass.
  2. Native path: dark materialization + flat processing validated.
  3. PixInsight path: dark template and flat template both validated.
  4. Output format toggle validated for XISF and FITS.
  5. Matching diagnostics regenerate cleanly post-process.

8. Code Quality Gates

  • Required: dotnet build and dotnet test green on FlatMaster.sln.
  • Required: No UI freeze regressions on long dark materialization stages.
  • Required: No writes into source dark library paths; output root only.
  • Recommended: Keep docs synchronized when changing matching priorities, thresholds, or output naming.

For algorithmic changes, use side-by-side PI minus native comparison with numeric stats before shipping.
