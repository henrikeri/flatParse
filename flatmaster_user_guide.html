1. Prerequisites

General

  • Windows 10 or 11
  • Input files in FITS, FIT, or XISF format
  • Sufficient disk space under selected output root

If using PixInsight mode

  • Valid PixInsight.exe path in Configuration tab
  • PixInsight script permissions enabled in your installation

3. Quick Workflow

  1. Open Configuration.
  2. Add at least one Flat Base Directory and one Dark Library Directory.
    • If running darks-only mode, only dark library roots are required.
  3. Choose output root and output format (XISF or FITS).
  4. Choose engine: Native or PixInsight.
  5. Go to Scan and Match and click Scan Flats + Darks.
  6. Wait for both scan bars to complete.
  7. Go to Matching Details to inspect selected dark per group (optional override).
  8. Go to Process and Report and click Start Processing.

Matching auto-runs after both scans complete, so diagnostics are ready when you open Matching Details.

4. Matching Details and Overrides

Columns explained

  • Exposure (s): flat group exposure.
  • Avg Flat Temp: average temperature of flats in group.
  • Selected Dark: current candidate chosen by priority logic.
  • Reason: priority code (P1-P8).
  • Temp Delta: delta between group average flat temp and selected dark temperature reference.
  • Override: force Auto or explicit candidate.

Override behavior

  • Auto uses the normal matching algorithm.
  • Manual selection forces a specific dark path for that group during processing.
  • Override lists include both explicit masters and loose-dark representatives that can be materialized into masters.

5. Processing Modes

Normal mode

  • Materializes required master darks first when loose darks are chosen.
  • Processes flat groups and creates master flats.
  • Can require darks or allow processing without dark subtraction when no match is found.

Darks-only mode

  • Builds master darks only.
  • Groups raw darks by exposure and temperature clusters (default tolerance +/-1.0 degC).
  • Does not process flats.

Bias fallback guard settings

Two settings control when darks are allowed to beat bias fallback:

  • Temperature delta limit (degC)
  • Exposure delta limit (seconds)

If absolute dark deltas exceed limits and bias exists, bias can be selected.

6. Output Structure and Naming

Master darks

OutputRoot\Master\Darks\25s\-10degC\MasterDark_25s_-10degC.xisf
OutputRoot\Master\Darks\25s\-10degC\MasterDark_25s_-10degC.fits

Master flats

  • Native engine naming includes bin token, for example: MasterFlat_2025-09-12_BLUE_Bin1_10.000s.xisf
  • PixInsight template naming omits bin token, for example: MasterFlat_2025-09-12_BLUE_10s.xisf

Output mode

  • Replicated separate tree writes under selected output root.
  • Optional cleanup removes calibrated flat intermediates after master creation.

7. Progress and Logs

  • Scan tab shows two dedicated progress bars (flats and darks) with folder/file counters.
  • Status summary reports final counts after scan completion.
  • Process tab log is the authoritative runtime trace for materialization and engine stages.
  • Session log path is shown at startup and can be used for support/debugging.

8. Troubleshooting

Common issues

SymptomLikely causeAction
No match and bias selected unexpectedly Strict temperature gate or bias-guard thresholds too tight Review matching reason, then tune delta limits in Configuration
No darks found during scan Wrong root path or unsupported file extensions Validate dark root and file types
PixInsight processing fails to start Invalid executable path or environment issue Re-browse executable in Configuration and retry
Processing appears stalled in long dark stage Large dark integration workload Monitor detailed stage logs in Process tab

Support bundle recommendation

When reporting issues, include:

  • Session log file path and full log content
  • Selected mode (Native or PixInsight)
  • Output format
  • One representative flat path and selected dark reason from Matching Details

For deterministic debugging, rerun with unchanged selections and an empty output root.
